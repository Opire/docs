# Einführung

## Was ist Opire?

Opire ist eine **Belohnungsplattform für Softwareentwickler**.

Dieses Projekt entstand mit <u>einem klaren Ziel</u>: dass **jeder seinen Lebensunterhalt mit Open-Source-Software verdienen kann**. Dies umfasst Programmierer, die sich auf die Entwicklung von Software und die Verbesserung von Open-Source-Code konzentrieren möchten, sowie Besitzer und Betreuer von Open-Source-Code, die mit ihren Projekten zur Community beitragen.

Mit Opire **kann jeder Belohnungen für Open-Source-Projekte erstellen** und seine Community erweitern, während Entwickler Probleme lösen und die zugehörigen Belohnungen **verdienen können**.

## Wie funktioniert es?

Opire besteht aus zwei Komponenten: einer [Bot](/overview/install-bot) (die auf GitHub installiert werden soll) und einer [Webplattform](https://app.opire.dev).

Die Bot ermöglicht die Verwendung verschiedener [Befehle](/overview/commands), die direkt in GitHub integriert sind, um mit der Plattform zu interagieren: Belohnungen erstellen, diese beanspruchen und mehr. Dies ermöglicht einen komfortableren und integrierten Workflow direkt im Arbeitsbereich der Entwickler.

Belohnungen können auch von der Webplattform aus erstellt und beansprucht werden, die sich auch auf deren Visualisierung und Verwaltung konzentriert. Unter anderem ermöglicht es eine erweiterte Suche (nach Technologie, Preis...), die Zahlung für bereits beanspruchte Belohnungen, die Historie der von Ihnen gelösten Belohnungen oder die Konfiguration Ihres Profils.

Für einen detaillierten Blick auf den Lebenszyklus der Belohnung werfen Sie einen Blick auf diesen [Abschnitt](/rewards/lifecycle).

### Vorteile für Open-Source-Betreuer

#### Sie verlieren nichts
<DimmedText>Unsere [Bot](/overview/install-bot) ist vollständig kostenlos. Sie verlieren nichts, indem Sie sie installieren (nur wenige Sekunden!), und Sie können sofort davon profitieren.</DimmedText>

#### Beliebtheit gewinnen
<DimmedText>In Ihren Projekten erstellte Belohnungen werden Entwickler anziehen, die daran interessiert sind, sie zu beanspruchen. Dies bringt Ihr Projekt ins Rampenlicht für großartige Fachleute.</DimmedText>

#### Robustere und vollständigere Software
<DimmedText>Indem Sie die Erstellung von Belohnungen für Ihre Probleme ermöglichen, schreitet Ihr Projekt schnell voran und verbessert sich. Dieser beschleunigte Entwicklungszyklus führt zu robusterer und widerstandsfähigerer Software, die sowohl Ihren Benutzern als auch Ihrem Ruf zugutekommt.</DimmedText>

#### Erweitern Sie Ihre Community
<DimmedText>Die Erlaubnis zur Erstellung von Belohnungen stärkt nicht nur Ihre Community, sondern fördert auch eine Kultur der Zusammenarbeit und Teilnahme an Ihrem Projekt.</DimmedText>

#### Halten Sie Ihr Projekt am Leben (auch wenn Sie keine Zeit haben)
<DimmedText>Wenn Sie aufgrund Ihrer Umstände nicht so viel Zeit wie gewünscht für die Pflege Ihres Projekts aufwenden können, öffnen Sie die Türen für andere Entwickler, die Ihnen helfen möchten.</DimmedText>

### Vorteile für Personen/Organisationen, die Belohnungen erstellen

#### Halten Sie Ihr Team auf das Wesentliche fokussiert
<DimmedText>
Sie können einen strategischen Vorteil erlangen, indem Sie Änderungen an Drittanbieterbibliotheken anfordern, ohne die Ressourcen Ihres Entwicklerteams zu gefährden. Diese Flexibilität ermöglicht es Ihrem Team, sich auf Bereiche zu konzentrieren, in denen es Mehrwert schaffen kann.
</DimmedText>

#### Verwenden Sie weiterhin Ihre Lieblingsbibliotheken
<DimmedText>
Sie müssen nicht aufhören, diese Bibliothek zu verwenden, die Sie so sehr lieben. Dieser lästige Fehler (dem niemand Beachtung geschenkt hat) wird Sie nicht aufhalten: Erstellen Sie eine Belohnung für einen Entwickler, um Ihr Problem zu lösen.
</DimmedText>

#### Schnellere Problemlösung
<DimmedText>
Ihre Projektabhängigkeiten verbessern sich schneller, indem Sie Entwickler dazu anregen, die für Sie relevanten Probleme zu lösen.
</DimmedText>

#### Unterstützen Sie Open-Source
<DimmedText>
Haben Sie das Gefühl, dass die Welt eine Schuld gegenüber Tausenden von Open-Source-Projekten hat? Geben Sie das Gefallen zurück, indem Sie zu einem Ihrer Lieblingsprojekte beitragen: [Erstellen Sie eine Belohnung](/overview/commands#create-reward) für eines seiner Probleme oder [geben Sie ein Trinkgeld](/overview/commands#tip-a-user) an seine Schöpfer.
</DimmedText>
