# Introdução

## O que é Opire?

Opire é uma **plataforma de recompensas para desenvolvedores de software**.

Este projeto nasce com <u>um objetivo claro</u>: que **qualquer pessoa possa ganhar a vida com software de código aberto**. Isso inclui programadores que desejam se concentrar no desenvolvimento de software e na melhoria do código-fonte aberto, bem como proprietários e mantenedores de código-fonte aberto que contribuem para a comunidade com seus projetos.

Com o Opire, **qualquer pessoa pode criar recompensas para projetos de código aberto** e aumentar sua comunidade, enquanto os desenvolvedores podem resolver problemas e **ganhar as recompensas associadas**.

## Como funciona?

O Opire consiste em dois componentes: um [bot](/overview/install-bot) (para instalar no GitHub) e uma [plataforma web](https://app.opire.dev).

O aplicativo permite o uso de diferentes [comandos](/overview/commands) diretamente integrados ao GitHub para interagir com a plataforma: criar recompensas, reivindicá-las e muito mais.

A plataforma web se concentra na visualização e gerenciamento de recompensas. Entre outras funcionalidades, permite busca avançada (por tecnologia, preço, etc.), pagamento de recompensas reivindicadas, histórico das resolvidas, configuração de perfil e muito mais.

Para uma visão detalhada do ciclo de vida da recompensa, confira esta [seção](/rewards/lifecycle).

### Benefícios para mantenedores de código aberto

#### Você não perde nada

<DimmedText>
  Nosso [aplicativo](/overview/install-bot) é totalmente gratuito. Você não
  perde nada ao instalá-lo (apenas alguns segundos!) E pode se beneficiar dele
  imediatamente.
</DimmedText>

#### Ganhar dinheiro

<DimmedText>
  Se alguém de fora do seu projeto criar uma recompensa dentro dele, você
  [receberá %](/rewards/pricing#percentage-for-code-owner) quando a recompensa
  for paga. Este pode ser um incentivo valioso para manter e melhorar seu
  projeto de código aberto.
</DimmedText>

#### Ganhar popularidade

<DimmedText>
  As recompensas criadas em seus projetos atrairão desenvolvedores interessados
  em reivindicá-las. Isso coloca seu projeto em destaque para grandes
  profissionais.
</DimmedText>

#### Software mais robusto e completo

<DimmedText>
  Ao possibilitar a criação de recompensas para seus problemas, seu projeto
  avança e melhora rapidamente. Esse ciclo de desenvolvimento acelerado resulta
  em software mais robusto e resiliente, beneficiando tanto seus usuários quanto
  sua reputação.
</DimmedText>

#### Expanda sua comunidade

<DimmedText>
  Permitir a criação de recompensas não apenas fortalece sua comunidade, mas
  também promove uma cultura de colaboração e participação em seu projeto.
</DimmedText>

#### Mantenha seu projeto vivo (mesmo que você não tenha tempo)

<DimmedText>
  Se, devido às suas circunstâncias, você não puder dedicar tanto tempo quanto
  gostaria para manter seu projeto, abra as portas para que outros
  desenvolvedores o ajudem.
</DimmedText>

### Benefícios para indivíduos/organizações que criam recompensas

#### Mantenha sua equipe focada no que importa

<DimmedText>
  Você pode obter uma vantagem estratégica solicitando modificações em
  bibliotecas de terceiros sem comprometer os recursos da sua equipe de
  desenvolvedores. Essa flexibilidade permite que sua equipe se concentre em
  áreas onde pode agregar valor.
</DimmedText>

#### Continue usando suas bibliotecas favoritas

<DimmedText>
  Você não precisa parar de usar aquela biblioteca que tanto ama. Esse bug chato
  (ao qual ninguém prestou atenção) não vai te impedir: crie uma recompensa para
  um desenvolvedor resolver seu problema.
</DimmedText>

#### Resolução mais rápida de problemas

<DimmedText>
  As dependências do seu projeto melhorarão mais rapidamente, incentivando os
  desenvolvedores a resolver os problemas que são importantes para você.
</DimmedText>

#### Ajude o código aberto a subir de nível

<DimmedText>
  Você já se deparou com um projeto de código aberto que parece promissor, mas
  não é tão sofisticado quanto você gostaria? Crie recompensas para que outros

</DimmedText>

#### Suporte de código aberto

<DimmedText>
  Você acha que o mundo tem uma dívida com milhares de projetos de código
  aberto? Retribua o favor contribuindo para um de seus projetos favoritos:
  [crie uma recompensa](/overview/commands#create-reward) por um de seus
