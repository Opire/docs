# Primeros pasos

## Como codeowner

1. Instalar la [app](https://docs.opire.dev/es/overview/installations)

<span style={{ color: 'grey'}}>Esta app permite a cualquier usuario usar los comandos de Opire en los repositorios donde esté instalada, pudiendo interactuar con la plataforma de Opire sin tener que salir de tu repositorio de GitHub.</span>
<br/><br/>
<span style={{ color: 'grey'}}>Te recomendamos que notifiques a tu comunidad de que estás usando Opire para que ellos mismos puedan crear las recompensas o estar atentos para resolver las issues. Para ello, puedes modificar el `README.md` de los repositorios, comunicarlo en el foro de la comunidad, avisar en Discord, etc...</span>

2. Crear tu primera recompensa (o deja que otros la creen por ti) usando [nuestros comandos](https://docs.opire.dev/es/overview/commands)

<span style={{ color: 'grey'}}>Si vas a crear rewards en tus propios repositorios, revisa la sección ["Como creador de recompensas"](https://docs.opire.dev/es/overview/getting-started#como-creador-de-recompensas).</span>

## Como creador de recompensas

1. Crear tu primera recompensa usando [nuestros comandos](https://docs.opire.dev/es/overview/commands)

<span style={{ color: 'grey'}}>Con Opire **cualquiera puede crear recompensas** en repositorios que tengan instalada la app. Se pueden empezar a ofrecer recompensas [desde $20](https://docs.opire.dev/es/rewards/pricing). Ademas de añadir recompensas a las issues, cualquiera puede agradecer el esfuerzo de algún usuario con propinas [desde $5](https://docs.opire.dev/es/tips/pricing).</span>
<br/><br/>
<span style={{ color: 'grey'}}>Una issue puede tener asociadas [múltiples recompensas](https://docs.opire.dev/es/rewards/workflow#multiples-recompensas), sin importar si es un mismo usuario el que las crea o son varios los que aportan diferentes cantidades: **a la hora del pago, cada usuario será responsable de pagar su parte**.</span>
<br/><br/>
<span style={{ color: 'grey'}}>Si una persona ajena a la organización del repositorio crea una recompensa, **la organización [recibirá un %](https://docs.opire.dev/es/rewards/pricing#porcentaje-del-codeowner)** cuando sea pagada al programador.</span>

2. Pagar cuando alguien reclame tus recompensas asociadas a la issue

<span style={{ color: 'grey'}}>Una vez el programador haya reclamado las recompensas de la issue y se revise que el cambio propuesto cumple con lo esperado, podrás pagar las recompensas que hayas creado para esa issue. En caso de que haya varios programadores que hayan reclamado la misma issue, podrás elegir a cuál pagarle. Es **responsabilidad del desarrollador asegurarse de que tiene correctamente configurada su cuenta** para poder recibir pagos.</span>
<br/><br/>
<span style={{ color: 'grey'}}>Es en este momento donde se realizará la transferencia de la recompensa junto al cobro de los [costes asociados](https://docs.opire.dev/es/rewards/pricing).</span>
<br/><br/>
<span style={{ color: 'grey'}}>Si [varias personas crean recompensas para la misma issue](https://docs.opire.dev/es/rewards/workflow#multiples-creadores), a cada creador se le cobrará la cantidad correspondiente de forma que sólo pagará la suma de las recompensas que haya creado.</span>

## Como developer

1. Asegurate de que tienes configurado el sistema de pagos(https://docs.opire.dev/es/payments)
2. Busca la reward que te interesa (¿hablar sobre los filtros del listado?)
3. Apuntate para comunicar a los demás sobre tu intención de resolver la issue
4. Haz PR y reclama la reward(https://docs.opire.dev/es/overview/commands)
5. Recibe tu reward
